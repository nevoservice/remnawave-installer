# Решение проблем

Типичные проблемы и их решения для remnawave-installer.

## Настройка DNS

**Проблема:** Установка не удаётся или SSL-сертификаты не получены.

**Решение:**
- Убедитесь, что DNS A-записи для всех доменов указывают на правильные IP-адреса **до** запуска скрипта
- Дождитесь распространения DNS (может занять до 48 часов, обычно 5-15 минут)
- Проверьте командой: `dig +short your-domain.com`

## Прокси Cloudflare

**Проблема:** Домен Selfsteal не работает с VLESS REALITY.

**Решение:**
- Для домена Selfsteal **отключите** проксирование Cloudflare (оранжевое облако)
- DNS-запись должна быть "DNS Only" (серое облако)
- Скрипт попытается это обнаружить, но проверьте вручную

## Лимиты Docker Hub

**Проблема:** Ошибка `You have reached your pull rate limit` при установке.

**Решения:**
1. Авторизуйтесь в Docker Hub:
   ```bash
   docker login
   ```
2. Подождите 6 часов для сброса лимита
3. Используйте другой IP-адрес или VPN

## Конфликты портов

**Проблема:** Сервисы не запускаются, порты уже заняты.

**Решение:**
- Скрипт следует запускать на "чистом" сервере
- Остановите конфликтующие сервисы:
  ```bash
  sudo systemctl stop nginx apache2
  sudo lsof -i :80 -i :443
  ```

## Проблемы с файрволом

**Проблема:** Сервисы работают, но недоступны извне.

**Решение:**
- Скрипт автоматически настраивает UFW
- При использовании другого файрвола откройте порты вручную:
  - 80/tcp, 443/tcp (для Caddy)
  - Ваш SSH-порт
  - 2222/tcp (если нужно для связи с нодой)

## Потеря доступа к панели (Всё-в-одном)

**Проблема:** Панель недоступна после остановки Xray ноды.

**Причина:** В режиме Всё-в-одном Caddy получает трафик через fallback Xray.

**Решения:**
1. **Экстренный доступ** (через меню скрипта): Открывает прямой доступ на порту 8443
2. **Ручное исправление**: Отредактируйте Caddyfile, замените `127.0.0.1` на `0.0.0.0` и перезапустите Caddy

## Управление сервисами

### Проверка статуса
```bash
cd /opt/remnawave  # или /opt/remnanode
docker compose ps
```

### Просмотр логов
```bash
make logs
# или
docker compose logs -f
```

### Перезапуск сервисов
```bash
make restart
```

## Потеря учётных данных

**Расположение:** `/opt/remnawave/credentials.txt`

**Если удалены:** Некоторые значения можно посмотреть в файлах `.env`, но пароли, возможно, придётся сбросить через CLI:
```bash
# Доступ к CLI Remnawave
docker exec -it remnawave /bin/sh
```

## Проблемы с обновлением

**Проблема:** Обновление зависло или не удалось.

**Решение:**
1. Проверьте, что Docker daemon работает
2. Проверьте интернет-соединение
3. Ручное обновление:
   ```bash
   cd /opt/remnawave
   docker compose pull
   docker compose up -d --force-recreate
   ```
